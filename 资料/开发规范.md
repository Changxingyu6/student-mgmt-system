# 学生管理系统开发规范

## 1. 目录职责

| 目录 | 职责 |
|------|------|
| `api/` | 路由定义、参数校验、响应格式化 |
| `services/` | 业务逻辑、事务管理 |
| `dao/` | 数据库 CRUD 操作 |
| `model/` | SQLAlchemy 模型定义 |
| `scheme/` | Pydantic 验证模型 |
| `deps.py` | 依赖注入函数 |
| `utils/` | 通用工具函数 |

## 3. API 开发规范

```python
# 路由定义
router = APIRouter(prefix="/students", tags=["学生管理"])

# HTTP 方法对应
GET /students      → 查询列表
GET /students/{id} → 查询单个
POST /students     → 新增
PUT /students/{id} → 更新
DELETE /students/{id} → 删除

# 响应格式
return format_response(data=data, message="success")
```

## 4. 数据库操作

```python
# 使用 SQLAlchemy ORM
def get_student(db: Session, student_id: int):
    return db.query(Student).filter(Student.id == student_id).first()

# 禁止拼接 SQL！
# ❌ 错误：db.execute(f"SELECT * FROM students WHERE id={id}")
# ✅ 正确：使用 ORM 或参数化查询
```

## 7. 代码风格

- 使用类型注解
- 函数/方法写文档字符串
- 保持函数单一职责
- 避免全局变量
