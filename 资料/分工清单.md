# 学生管理系统 - 5人分工清单

## 分工原则
每人负责完整模块开发（Model → DAO → Service → API），确保全流程经验。

---

## 成员A - 学生基本信息管理模块
| 任务 | 内容 |
|------|------|
| 数据库设计 | 设计 `students` 表结构 |
| Model | 定义 Student 模型 |
| DAO | 学生增删改查操作 |
| Service | 学生业务逻辑处理 |
| API | `/students` 路由实现 |

**字段**：学生编号、班级、姓名、籍贯、毕业院校、专业、入学时间、毕业时间、学历、顾问编号、年龄、性别

---

## 成员B - 成绩管理模块
| 任务 | 内容 |
|------|------|
| 数据库设计 | 设计 `scores` 表结构 |
| Model | 定义 Score 模型 |
| DAO | 成绩录入、修改、删除 |
| Service | 成绩业务逻辑 |
| API | `/scores` 路由实现 |

**字段**：学生编号、考核序次、成绩

---

## 成员C - 就业管理模块
| 任务 | 内容 |
|------|------|
| 数据库设计 | 设计 `employment` 表结构 |
| Model | 定义 Employment 模型 |
| DAO | 就业信息增删改查 |
| Service | 就业业务逻辑 |
| API | `/employment` 路由实现 |

**字段**：学生编号、就业开放时间、offer下发时间、公司名称、薪资

---

## 成员D - 班级+教师管理模块
| 任务 | 内容 |
|------|------|
| 数据库设计 | 设计 `classes`、`teachers` 表 |
| Model | 定义 Class、Teacher 模型 |
| DAO | 班级/教师增删改查 |
| Service | 班级/教师业务逻辑 |
| API | `/classes`、`/teachers` 路由 |

**班级字段**：班级编号、开课时间、班主任、授课老师  
**教师字段**：教师信息、带班信息

---

## 成员E - 统计分析+用户认证模块
| 任务 | 内容 |
|------|------|
| 数据库设计 | 设计 `users` 表 |
| Model | 定义 User 模型 |
| DAO | 用户认证相关操作 |
| Service | 统计分析逻辑、认证逻辑 |
| API | `/statistics`、`/auth` 路由 |

**统计功能**：年龄筛选、班级人数统计、成绩统计、就业统计

---

## 开发流程
1. 需求分析 → 数据库设计（集体讨论）
2. 各自实现负责模块（独立开发）
3. 代码整合与联调（集体协作）
4. 测试与修复（集体测试）

## 协作要点
- 统一规范：遵循 `开发规范.md`
- 数据库设计：先集体评审表结构
- 代码审查：提交前交叉检查
- 每日同步：简短站会同步进度
