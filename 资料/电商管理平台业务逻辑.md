# 电商管理平台业务逻辑说明

## 一、用户管理模块

### 1.1 用户信息新增
- **功能描述**：录入新用户账号信息
- **必填字段**：用户名、密码、手机号
- **可选字段**：昵称、邮箱、性别、头像
- **默认值**：用户等级=青铜，积分=0，状态=激活
- **业务规则**：
  - 用户名唯一性校验
  - 手机号格式校验（11位数字）
  - 邮箱格式校验
  - 密码需符合复杂度要求（至少8位，包含大小写字母、数字、特殊字符中的至少2种）

### 1.2 用户信息查询
- **功能描述**：支持多条件筛选查询用户列表
- **筛选条件**：
  - 用户ID（精确匹配）
  - 手机号（精确匹配）
  - 昵称（模糊查询）
  - 用户等级（枚举：青铜/白银/黄金/铂金/钻石）
  - 注册时间范围（起止时间）
  - 账号状态（激活/禁用/封禁）
- **返回字段**：用户ID、用户名、昵称、手机号、邮箱、性别、头像、等级、积分、状态、注册时间
- **分页支持**：支持页码和每页数量参数

### 1.3 用户信息修改
- **功能描述**：编辑用户基础信息
- **可修改字段**：昵称、手机号、邮箱、性别、头像、积分、等级、账号状态
- **业务规则**：
  - 修改手机号需验证新手机号格式
  - 修改邮箱需验证新邮箱格式
  - 积分可为正数或负数
  - 等级升级需满足积分条件（需配置规则）
  - 禁用用户后无法登录

### 1.4 用户逻辑删除
- **功能描述**：对注销/违规用户执行逻辑删除
- **业务规则**：
  - 将 `is_deleted` 字段设为 TRUE
  - 不物理删除数据，保留历史记录
  - 删除后用户无法登录
  - 查询时过滤已删除用户

## 二、商品管理模块

### 2.1 商品基础操作

#### 新增商品
- **必填字段**：商品编号、商品名称、分类ID、售价
- **可选字段**：原价、成本价、商品简介、主图、详情图、品牌、产地、库存数量、预警库存
- **业务规则**：
  - 商品编号唯一性校验
  - 售价必须大于0
  - 原价 >= 售价
  - 成本价 <= 售价
  - 库存数量默认为0
  - 预警库存默认为10
  - 上架状态默认为"在售"

#### 查询商品
- **筛选条件**：
  - 商品ID（精确匹配）
  - 商品名称（模糊查询）
  - 分类ID（精确匹配）
  - 价格区间（最低价~最高价）
  - 上下架状态（在售/下架）
- **排序支持**：按价格、销量、创建时间排序

#### 编辑商品
- **可修改字段**：所有商品字段（商品编号除外）
- **业务规则**：同新增商品

#### 下架/删除商品
- **下架**：将状态改为"下架"，商品仍可查询但不可购买
- **删除**：逻辑删除，将 `is_deleted` 设为 TRUE

### 2.2 商品分类管理
- **分类层级**：支持二级分类（一级分类、二级分类）
- **必填字段**：分类名称
- **可选字段**：父分类ID、排序、分类图标
- **业务规则**：
  - `parent_id=0` 表示一级分类
  - `parent_id>0` 表示二级分类，必须关联有效的一级分类
  - 分类名称在同一层级下唯一
  - 排序值越小越靠前
  - 禁用分类后，该分类下的商品不可购买

### 2.3 商品规格管理
- **规格类型**：颜色、尺寸、款式等
- **必填字段**：商品ID、规格名称、规格值
- **可选字段**：该规格库存、价格差异
- **业务规则**：
  - 每个商品可有多个规格维度（如：颜色+尺寸）
  - 同一规格名称下的规格值不能重复
  - 价格差异可为正数或负数
  - 删除规格需先清空该规格的库存

### 2.4 库存管理
- **库存增减**：支持手动增减库存数量
- **库存预警**：当库存低于预警值时，系统记录告警
- **查询预警**：查询所有库存低于预警值的商品
- **业务规则**：
  - 库存不能为负数
  - 下单时自动扣减库存
  - 取消订单时自动恢复库存

## 三、订单管理模块

### 3.1 订单录入与查询

#### 创建订单
- **触发方式**：用户从购物车下单
- **必填字段**：用户ID、收货人姓名、收货人电话、收货地址
- **自动生成**：订单编号、订单总金额、实付金额、下单时间
- **业务规则**：
  - 订单编号格式：ORD + 时间戳 + 随机数（如：ORD202406071234567890）
  - 订单总金额 = Σ(商品单价 × 数量)
  - 实付金额 = 订单总金额 - 优惠券抵扣金额
  - 初始状态：待确认（pending）
  - 初始支付状态：未支付（unpaid）
  - 初始物流状态：未发货（unshipped）
  - 校验库存：下单时校验商品库存是否充足

#### 查询订单列表
- **筛选条件**：
  - 订单编号（精确匹配）
  - 用户ID（精确匹配）
  - 下单时间范围
  - 订单状态（待确认/已确认/已发货/已签收/已完成/已取消/已关闭）
  - 支付状态（未支付/已支付/已退款）
  - 物流状态（未发货/已发货/已签收）
- **分页支持**：支持页码和每页数量参数

#### 查询订单详情
- **返回字段**：
  - 订单基础信息：订单编号、用户ID、总金额、实付金额、优惠券ID、抵扣金额
  - 收货信息：收货人姓名、电话、地址
  - 状态信息：订单状态、支付状态、物流状态
  - 时间信息：下单时间、支付时间、发货时间、签收时间、完成时间
  - 商品清单：商品名称、规格、单价、数量、小计金额
  - 备注：订单备注

### 3.2 订单状态操作

#### 状态流转规则
```
pending（待确认）
    ↓ 确认订单
confirmed（已确认）
    ↓ 发货
shipped（已发货）
    ↓ 签收
received（已签收）
    ↓ 完成
completed（已完成）
```

#### 取消/关闭订单
- **取消**：未支付订单可取消，状态改为 cancelled
- **关闭**：已支付订单可关闭（如超时未支付），状态改为 closed
- **业务规则**：
  - 取消订单后恢复库存
  - 关闭订单后不恢复库存

### 3.3 订单修改与删除
- **修改限制**：仅未支付订单可修改收货地址和备注
- **逻辑删除**：作废无效历史订单，将 `is_deleted` 设为 TRUE
- **业务规则**：
  - 已支付订单不可修改
  - 删除订单不影响商品库存

### 3.4 售后订单管理
- **售后类型**：退款、退货、换货
- **必填字段**：订单ID、售后类型、售后原因
- **可选字段**：退款金额、备注
- **审核状态**：待审核、已通过、已拒绝、已完成
- **业务规则**：
  - 退款金额不能超过订单实付金额
  - 管理员审核后更新状态
  - 已通过的退款订单自动退款
  - 退货订单需用户寄回商品后处理

## 四、购物车管理模块

### 4.1 购物车新增
- **必填字段**：用户ID、商品ID、购买数量
- **可选字段**：规格ID
- **业务规则**：
  - 用户ID + 商品ID + 规格ID 唯一
  - 重复添加同一商品则增加数量
  - 默认选中状态为 TRUE
  - 自动计算小计金额（单价 × 数量）

### 4.2 购物车查询
- **查询条件**：用户ID
- **返回字段**：
  - 购物车ID、商品ID、规格ID、数量、选中状态
  - 商品信息：商品名称、主图、单价
  - 小计金额

### 4.3 购物车修改
- **可修改字段**：购买数量、选中状态
- **业务规则**：
  - 数量必须大于0
  - 数量不能超过商品库存
  - 切换选中状态不影响其他商品

### 4.4 购物车删除
- **删除方式**：
  - 单条删除：删除指定购物车记录
  - 批量删除：删除多条购物车记录
  - 清空购物车：删除该用户的所有购物车记录
- **业务规则**：删除后不可恢复

---

## 五、支付&物流管理模块

### 5.1 支付记录管理
- **必填字段**：支付单号、订单ID、支付金额、支付方式
- **自动生成**：支付单号、支付时间
- **可选字段**：交易流水号
- **支付方式**：微信、支付宝、银行卡、余额
- **支付状态**：待支付、支付成功、支付失败、已退款
- **业务规则**：
  - 支付单号格式：PAY + 时间戳 + 随机数
  - 支付金额必须等于订单实付金额
  - 支付成功后更新订单支付状态和支付时间
  - 支付失败后订单状态保持不变

### 5.2 物流信息管理
- **必填字段**：订单ID、物流公司、物流单号
- **自动生成**：发货时间
- **可选字段**：物流轨迹、签收时间
- **业务规则**：
  - 发货时录入物流信息
  - 物流轨迹以JSON格式存储，包含物流节点更新
  - 签收时更新签收时间和订单状态
  - 支持根据订单ID或物流单号查询物流信息

## 六、营销活动管理模块

### 6.1 优惠券管理
- **必填字段**：券编号、券名称、面额、发放总量
- **可选字段**：使用门槛、有效期、使用状态
- **使用状态**：可用、已下架
- **业务规则**：
  - 优惠券编号唯一
  - 面额必须大于0
  - 使用门槛默认为0（无门槛）
  - 有效期可以是固定天数或指定日期
  - 已领数量不能超过发放总量
  - 下架后用户无法领取和使用

#### 领取优惠券
- **业务规则**：
  - 每个用户每张优惠券限领1张
  - 领取后记录到用户优惠券关联表
  - 增加优惠券的已领数量
  - 记录领取时间

#### 使用优惠券
- **业务规则**：
  - 订单金额必须满足使用门槛
  - 优惠券在有效期内
  - 每个订单只能使用一张优惠券
  - 使用后增加优惠券的已用数量
  - 记录使用时间和关联订单

### 6.2 满减/限时活动
- **活动类型**：折扣、限时抢购、满减
- **必填字段**：活动名称、活动类型、活动时间
- **可选字段**：优惠力度、满减门槛、参与商品
- **活动状态**：未开始、进行中、已结束、已禁用
- **业务规则**：
  - 活动时间必须包含开始时间和结束时间
  - 折扣活动：设置折扣比例（如0.8表示8折）
  - 满减活动：设置满减门槛和减免金额（如满100减20）
  - 限时抢购：设置活动价和库存限制
  - 活动结束后自动失效
  - 活动商品可设置专属活动价

## 七、数据统计分析模块

### 7.1 用户数据统计

#### 新增用户统计
- **功能**：查询指定时间段内新增用户数量
- **SQL逻辑**：
  ```sql
  SELECT COUNT(*) FROM users
  WHERE created_at BETWEEN ? AND ?
  AND is_deleted = FALSE
  ```

#### 活跃用户统计
- **功能**：查询指定时间段内有下单行为的用户数量
- **SQL逻辑**：
  ```sql
  SELECT COUNT(DISTINCT user_id) FROM orders
  WHERE created_at BETWEEN ? AND ?
  AND is_deleted = FALSE
  ```

#### 用户等级分布
- **功能**：统计各用户等级的人数
- **SQL逻辑**：
  ```sql
  SELECT user_level, COUNT(*) as count
  FROM users
  WHERE is_deleted = FALSE
  GROUP BY user_level
  ```

#### 男女用户数量
- **功能**：统计男女用户数量
- **SQL逻辑**：
  ```sql
  SELECT gender, COUNT(*) as count
  FROM users
  WHERE is_deleted = FALSE
  GROUP BY gender
  ```

#### 积分Top10用户
- **功能**：查询积分最高的10个用户
- **SQL逻辑**：
  ```sql
  SELECT username, nickname, points
  FROM users
  WHERE is_deleted = FALSE
  ORDER BY points DESC
  LIMIT 10
  ```

### 7.2 商品数据统计

#### 销量Top20商品
- **功能**：查询销量前20的商品
- **SQL逻辑**：
  ```sql
  SELECT g.id, g.goods_name, SUM(oi.quantity) as total_sales,
         SUM(oi.quantity * oi.price) as total_amount,
         c.category_name
  FROM goods g
  JOIN order_items oi ON g.id = oi.goods_id
  JOIN orders o ON oi.order_id = o.id
  JOIN goods_categories c ON g.category_id = c.id
  WHERE g.is_deleted = FALSE AND o.is_deleted = FALSE
  GROUP BY g.id
  ORDER BY total_sales DESC
  LIMIT 20
  ```

#### 分类商品统计
- **功能**：统计各分类的在售商品数量和平均售价
- **SQL逻辑**：
  ```sql
  SELECT c.category_name, COUNT(*) as goods_count,
         AVG(g.sale_price) as avg_price
  FROM goods_categories c
  LEFT JOIN goods g ON c.id = g.category_id AND g.status = 'on_sale' AND g.is_deleted = FALSE
  GROUP BY c.id
  ```

#### 库存预警商品
- **功能**：查询库存低于预警值的商品
- **SQL逻辑**：
  ```sql
  SELECT id, goods_name, stock, stock_warning
  FROM goods
  WHERE stock < stock_warning
  AND is_deleted = FALSE
  ```

### 7.3 订单&交易统计

#### 订单总量统计
- **功能**：按日/周/月统计订单总量和成交金额
- **SQL逻辑**：
  ```sql
  SELECT DATE(created_at) as date, COUNT(*) as order_count,
         SUM(pay_amount) as total_amount
  FROM orders
  WHERE is_deleted = FALSE AND pay_status = 'paid'
  GROUP BY DATE(created_at)
  ORDER BY date DESC
  ```

#### 退款订单统计
- **功能**：统计退款订单数量及占比
- **SQL逻辑**：
  ```sql
  SELECT
    COUNT(*) as total_orders,
    SUM(CASE WHEN pay_status = 'refunded' THEN 1 ELSE 0 END) as refunded_orders,
    SUM(CASE WHEN pay_status = 'refunded' THEN 1 ELSE 0 END) / COUNT(*) * 100 as refund_rate
  FROM orders
  WHERE is_deleted = FALSE
  ```

#### 支付方式统计
- **功能**：统计各支付方式的订单数量和交易金额
- **SQL逻辑**：
  ```sql
  SELECT pay_method, COUNT(*) as order_count,
         SUM(amount) as total_amount
  FROM payments
  WHERE pay_status = 'success'
  GROUP BY pay_method
  ORDER BY total_amount DESC
  ```

#### 异常订单查询
- **功能**：查询未付款超时订单、长时间未发货订单
- **SQL逻辑**：
  ```sql
  -- 未付款超时订单（超过30分钟）
  SELECT id, order_no, created_at
  FROM orders
  WHERE pay_status = 'unpaid'
  AND TIMESTAMPDIFF(MINUTE, created_at, NOW()) > 30
  AND is_deleted = FALSE

  -- 长时间未发货订单（已支付超过24小时）
  SELECT id, order_no, paid_at
  FROM orders
  WHERE pay_status = 'paid'
  AND logistics_status = 'unshipped'
  AND TIMESTAMPDIFF(HOUR, paid_at, NOW()) > 24
  AND is_deleted = FALSE
  ```

### 7.4 营销数据统计

#### 优惠券核销率
- **功能**：统计各优惠券的领取数量、使用数量、核销率
- **SQL逻辑**：
  ```sql
  SELECT c.coupon_name, c.received_count, c.used_count,
         CASE WHEN c.received_count > 0
              THEN c.used_count / c.received_count * 100
              ELSE 0 END as usage_rate
  FROM coupons c
  WHERE c.status = 'active'
  ```

#### 活动效果统计
- **功能**：统计每场营销活动带来的订单量和成交额
- **SQL逻辑**：
  ```sql
  SELECT a.activity_name, COUNT(DISTINCT o.id) as order_count,
         SUM(o.pay_amount) as total_amount
  FROM activities a
  JOIN activity_goods ag ON a.id = ag.activity_id
  JOIN order_items oi ON ag.goods_id = oi.goods_id
  JOIN orders o ON oi.order_id = o.id
  WHERE a.status = 'ended' AND o.pay_status = 'paid'
  GROUP BY a.id
  ORDER BY total_amount DESC
  ```

## 八、店铺管理模块

### 8.1 店铺信息管理

#### 创建店铺
- **功能描述**：店主注册并创建店铺
- **必填字段**：店铺名称、店铺编号、店主用户ID
- **可选字段**：Logo、横幅、店铺简介
- **默认值**：状态=激活，评分=5.0
- **业务规则**：
  - 店铺编号唯一性校验
  - 店主必须是已注册用户
  - 店铺名称长度限制

#### 查询店铺
- **筛选条件**：
  - 店铺ID（精确匹配）
  - 店铺名称（模糊查询）
  - 店主ID（精确匹配）
  - 店铺状态（激活/关闭/暂停）
- **返回字段**：店铺ID、店铺名称、店铺编号、店主信息、Logo、状态、评分、创建时间
- **分页支持**：支持页码和每页数量参数

#### 修改店铺信息
- **可修改字段**：店铺名称、Logo、横幅、店铺简介、状态
- **业务规则**：
  - 店铺编号不可修改
  - 状态变更需记录操作日志

#### 删除店铺
- **逻辑删除**：将状态改为"关闭"，保留历史数据

### 8.2 店铺商品管理
- **功能描述**：管理店铺上架的商品
- **业务规则**：
  - 商品必须先存在于商品表
  - 同一商品可上架到多个店铺
  - 支持设置排序和推荐标记
  - 删除店铺商品不影响商品表数据

### 8.3 店铺评分
- **功能描述**：用户购买商品后评价店铺
- **业务规则**：

## 十一、日志管理模块

  - 保留最近180天操作日志

## 十二、通用约束

1. **逻辑删除**：所有删除操作优先使用逻辑删除，保留业务历史数据
2. **RESTful规范**：接口统一遵循RESTful设计规范，保证命名、请求方式统一
3. **数据校验**：数据入参强制使用Pydantic模型校验，拦截非法参数
4. **响应格式**：接口响应格式统一，包含状态码、提示信息、业务数据
5. **时间格式**：统一使用DATETIME类型，格式为 `YYYY-MM-DD HH:MM:SS`
6. **金额格式**：统一使用DECIMAL(10,2)，保留两位小数
7. **分页参数**：page（页码，从1开始）、limit（每页数量，默认10）
