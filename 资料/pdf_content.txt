电商管理平台需求文档
1. 项目概述
本项目基于FastAPI开发一套电商后台管理平台，采用 RESTful API 设计规范，支持前后端分离架构。系统核心实
现商品管理、订单管理、用户管理、购物车管理、支付物流管理、营销活动管理六大业务模块，同时配套数据统计
分析功能，满足电商日常运营、数据查询、业务管控等需求，接口自动生成 Swagger 在线文档，保障接口调试与
对接效率。
2. 功能需求
2.1 用户管理模块
实现平台会员、账号信息全生命周期管理，支持基础信息维护与条件查询。
1. 用户信息新增：录入用户账号、昵称、手机号、邮箱、注册时间、用户等级、默认收货地址、账号状态、会
员积分、性别、头像等信息。
2. 用户信息查询：支持按用户 ID、手机号、昵称、用户等级、注册时间范围、账号状态多条件筛选查询用户列
表。
3. 用户信息修改：编辑用户昵称、手机号、地址、积分、等级、账号状态等内容。
4. 用户逻辑删除：对注销 / 违规用户执行逻辑删除，保留历史数据。
2.2 商品管理模块
负责商品上下架、分类、规格、库存等核心商品数据管理。
1. 商品基础操作：新增商品（商品编号、商品名称、分类 ID、售价、原价、成本价、商品简介、详情图、主
图、品牌、产地、上架状态、库存数量、预警库存）；根据商品 ID、名称、分类、价格区间、上下架状态查
询商品；编辑商品全部信息；下架 / 删除商品。
2. 商品分类管理：维护一级 / 二级商品分类（分类编号、分类名称、排序、分类图标、创建时间、状态），支持
增、改、查、禁用操作。
3. 商品规格管理：管理商品规格（颜色、尺寸、款式），绑定对应商品，支持规格新增、修改、删除。
4. 库存管理：单独维护商品库存，支持手动增减库存、设置库存预警，查询库存不足商品。
2.3 订单管理模块
覆盖订单全流程管控，包含订单创建、状态流转、信息查询与售后处理。
1. 订单录入与查询：用户下单自动生成订单，支持按订单编号、用户 ID、下单时间、订单状态、支付状态、物
流状态筛选订单列表；查询单条订单详情（商品清单、实付金额、收货地址、支付方式、下单时间、发货时
间、完成时间）。
2. 订单状态操作：订单确认、发货、签收、完成、取消、关闭等状态变更。
3. 订单修改与删除：仅支持未支付订单修改收货地址、备注；作废无效历史订单（逻辑删除）。
4. 售后订单管理：记录退款、退货、换货订单，录入售后原因、售后金额、审核状态，支持售后审核、查询、
修改。
2.4 购物车管理模块
管理用户购物车数据，关联用户与商品。
1. 购物车新增：用户添加商品至购物车（用户 ID、商品 ID、购买数量、选中状态、添加时间）。
2. 购物车查询：根据用户 ID 查询个人购物车列表，展示商品信息、单价、小计金额。
3. 购物车修改：修改商品购买数量、切换商品选中状态。
4. 购物车删除：单条 / 批量删除购物车商品，清空个人购物车。
2.5 支付 \& 物流管理模块
管控订单支付记录、物流轨迹、配送信息。
1. 支付记录管理：记录支付单号、关联订单 ID、支付金额、支付方式、支付时间、支付状态、交易流水号；支
持按订单、支付时间查询支付记录，修改异常支付状态。
2. 物流信息管理：录入物流公司、物流单号、发货时间、物流轨迹、签收时间；根据订单 ID、物流单号查询物
流信息，更新物流节点。
2.6 营销活动管理模块
负责平台促销活动配置与管理，包含优惠券、限时活动、满减活动。
1. 优惠券管理：创建优惠券（券编号、券名称、面额、使用门槛、有效期、发放总量、已领数量、使用状
态）；查询、编辑、下架优惠券；查看用户领券、用券记录。
2. 满减 / 限时活动：配置活动规则、活动时间、参与商品、优惠力度；活动启停、修改、删除；查询活动参与订
单。
2.7 数据统计分析模块
2.7.1 用户数据统计
1. 查询指定时间段内新增用户、活跃用户明细。
2. 统计各用户等级人数、男女用户数量。
3. 统计会员积分 Top10 用户信息。
2.7.2 商品数据统计
1. 查询销量前 20 的商品名称、销量、销售额、所属分类。
2. 统计各商品分类的在售商品数量、平均售价。
3. 查询库存低于预警值的所有商品。
2.7.3 订单 \& 交易统计
1. 按日 / 周 / 月统计订单总量、成交金额、退款订单数量及占比。
2. 统计各支付方式的订单数量与交易金额，按金额降序排序。
3. 查询未付款超时订单、长时间未发货订单明细。
2.7.4 营销数据统计
1. 统计各优惠券领取数量、使用数量、核销率。
2. 统计每场营销活动带来的订单量、成交额。
3. 非功能需求
3.1 技术栈
后端框架：FastAPI
数据库：MySQL
数据校验：Pydantic
接口文档：内置 Swagger 自动生成 API 在线文档，支持在线调试接口
3.2 通用约束
1. 所有删除操作优先使用逻辑删除，保留业务历史数据。
2. 接口统一遵循 RESTful 设计规范，保证命名、请求方式统一。
3. 数据入参强制使用 Pydantic 模型校验，拦截非法参数。
4. 接口响应格式统一，包含状态码、提示信息、业务数据。
4. API 设计概要
所有接口遵循 RESTful 规范，兼顾前端调用便捷性与接口执行效率。
4.1 用户管理接口
GET /users ：获取用户列表（支持多条件筛选）
POST /users ：新增用户
GET /users/{user_id} ：获取单个用户详情
PUT /users/{user_id} ：修改用户信息
DELETE /users/{user_id} ：逻辑删除用户
4.2 商品管理接口
GET /goods ：获取商品列表
POST /goods ：新增商品
GET /goods/{goods_id} ：获取单商品详情
PUT /goods/{goods_id} ：编辑商品信息
DELETE /goods/{goods_id} ：删除商品
GET /goods/category ：获取商品分类列表
POST /goods/category ：新增分类
PUT /goods/category/{cate_id} ：修改分类
4.3 订单管理接口
GET /orders ：获取订单列表（多条件筛选）
GET /orders/{order_id} ：获取订单详情
PUT /orders/{order_id}/status ：修改订单状态
GET /orders/aftersale ：查询售后订单
PUT /orders/aftersale/{id} ：审核售后订单
4.4 购物车接口
GET /cart/{user_id} ：获取用户购物车
POST /cart ：添加商品到购物车
PUT /cart/{cart_id} ：修改购物车商品数量 / 选中状态
DELETE /cart/{cart_id} ：删除购物车单条数据
4.5 支付 \& 物流接口
GET /pay ：查询支付记录
GET /logistics/{order_id} ：查询订单物流信息
POST /logistics ：录入物流信息
PUT /logistics/{logi_id} ：更新物流轨迹
4.6 营销活动接口
GET /coupon ：获取优惠券列表
POST /coupon ：新增优惠券
GET /activity ：查询营销活动列表
POST /activity ：创建营销活动
4.7 数据统计接口
统一路径前缀 /statistics
GET /statistics/user ：用户数据统计
GET /statistics/goods ：商品销量、库存统计
GET /statistics/order ：订单交易数据统计
GET /statistics/marketing ：营销活动数据统计
4.8 其他模块接口
店铺管理、管理员账号、系统配置等模块接口可根据业务自行定义路径与请求方式。
5. 后续扩展与迭代规划
1. 新增店铺管理、供应商管理、角色权限管理模块。
2. 增加用户登录、账号认证、权限控制模块，实现接口鉴权。
3. 开发配套前端管理页面、用户商城页面。
4. 拓展更多精细化数据报表、可视化查询接口。
5. 接入消息推送、智能推荐、AI 客服等智能化功能。
6. 对接第三方物流、第三方支付平台接口。
6. 开发与答辩要求
1. 以小组为单位完成需求研讨、方案确认与代码开发。
2. 标准开发流程：需求分析 → 数据库设计 → 系统概要设计 → 功能编码开发 → 功能测试。
3. 答辩要求：每组安排专人讲解项目需求、整体架构、功能模块；随机抽查组员解读代码、讲解数据库设计、
接口设计思路。
4. 答辩提问包含本项目设计、代码实现，同时抽查前期基础知识内容（占比约 20%）。
